# NaviDeskApp Requirements

## Goal

NaviDeskApp の目的は、社員が正しい業務情報に最短で到達でき、運用担当者が安全かつ継続的に改善できる社内向けマニュアル Web アプリケーションを提供することです。

## Core requirements

### 1. 情報到達

- 社員はホーム検索から記事と FAQ を横断検索できる
- 社員はタスクハブ、カテゴリ、クイックリンクから業務導線へ到達できる
- ゼロ件検索時は代替導線を提示する

### 2. 情報信頼性

- 公開コンテンツは承認済みのみ表示する
- AI 案内は社内コンテンツを根拠にした回答のみ返す
- 根拠が弱い場合は断定せず候補のみ返す

### 3. 運用統制

- 記事、FAQ、お知らせ、クイックリンクの管理操作を行える
- 記事、FAQ、お知らせは承認申請、承認、差し戻しを管理できる
- 監査ログで作成、更新、削除、承認系操作を追跡できる

### 4. 継続改善

- 検索失敗テーマを集計し、情報不足を見える化する
- 更新期限が近い、または古いコンテンツをレビュー優先キューで確認できる
- 記事詳細から承認情報、レビューコメント、変更履歴を確認できる

### 5. セキュリティ・権限

- ロール別に閲覧可否を制御する
- ロールはサーバー側で決定する
- 非公開コンテンツを一般閲覧者へ送信しない

## Current implementation status

### 実装済み

- ホーム検索、FAQ検索、AI案内
- タスクハブ、カテゴリ導線、クイックリンク導線
- 承認フロー付き管理画面
- 監査ログ画面
- 検索失敗テーマ可視化
- 更新期限レビュー優先キュー
- 記事詳細の承認・履歴表示

### 本番移行時に必要

- SSO / OIDC 認証
- データベース永続化
- サーバーサイド監査ログ
- サーバーサイド検索分析保存
- 差分比較 UI と履歴リビジョン管理
